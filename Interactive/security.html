Azure Firewall

Azure Firewall Overview

Azure Firewall is a cloud-based network security service that protects your Azure Virtual Network resources. It's a stateful firewall as a service with built-in high availability and unrestricted cloud scalability.

Core Capabilities

  • Stateful packet inspection
  • Application and network rules
  • Threat intelligence-based filtering
  • Outbound SNAT support
  • Inbound DNAT support
  • Centralized policy management

Security Features

  • Application FQDN filtering
  • Network traffic filtering rules
  • Threat intelligence feeds
  • TLS inspection (Premium)
  • IDPS signatures (Premium)
  • URL filtering (Premium)

Firewall Policies and Rules

Rule Types

Application Rules

Control outbound HTTP/HTTPS traffic using FQDNs

  • Allow/deny specific URLs
  • Support for wildcards
  • TLS inspection (Premium)
Network Rules

Control traffic using 5-tuple (IP, port, protocol)

  • Inbound and outbound rules
  • TCP, UDP, ICMP, Any protocols
  • IP ranges and ports
NAT Rules

DNAT (Destination NAT) for inbound connections

  • Port forwarding
  • IP address translation
  • Protocol translation

Rule Processing Order

1. DNAT Rules
Inbound NAT rules processed first
2. Network Rules
Network rules processed by priority
3. Application Rules
Application rules processed by priority
4. Default Deny
Implicit deny if no rules match
# Create firewall policy az network firewall policy create \ --name MyFirewallPolicy \ --resource-group MyResourceGroup \ --location eastus # Add application rule az network firewall policy rule-collection-group collection rule add \ --collection-name AppRuleCollection \ --name AllowMicrosoft \ --policy-name MyFirewallPolicy \ --resource-group MyResourceGroup \ --rule-type ApplicationRule \ --target-fqdns *.microsoft.com \ --source-addresses 10.0.0.0/16 \ --protocols Http Https

Network Security Groups (NSGs)

Network Security Groups (NSGs) are Azure's basic network security feature that acts as a simple firewall, allowing or denying traffic based on 5-tuple rules.

NSG Components

  • Security Rules: Allow or deny traffic
  • Default Rules: Built-in rules for basic security
  • Application Security Groups: Group VMs by security requirements
  • Service Tags: Represent groups of IP addresses

Service Tags

• VirtualNetwork
• AzureLoadBalancer
• Internet
• AzureTrafficManager
• Storage
• Sql
• AzureKeyVault
• AzureContainerRegistry

NSG Rule Structure

Component Description Example
Name Unique rule name AllowHTTP
Priority Processing order (100-4096) 100
Source Source IP or range 10.0.0.0/16
Source Port Source port or range *
Destination Destination IP or range *
Destination Port Destination port or range 80
Protocol TCP, UDP, or Any TCP

# Create NSG az network nsg create \ --name MyNsg \ --resource-group MyResourceGroup \ --location eastus # Add security rule az network nsg rule create \ --name AllowHTTPS \ --nsg-name MyNsg \ --resource-group MyResourceGroup \ --priority 100 \ --direction Inbound \ --access Allow \ --protocol Tcp \ --source-address-prefix '*' \ --source-port-range '*' \ --destination-address-prefix '*' \ --destination-port-range 443

DDoS Protection

Azure DDoS Protection helps protect your Azure resources from distributed denial-of-service (DDoS) attacks. It provides always-on monitoring and automatic attack mitigation.

DDoS Protection Tiers

Basic (Free)
  • Always-on traffic monitoring
  • Automatic attack mitigation
  • Basic L3/4 protection
  • Microsoft managed
Standard (Paid)
  • Enhanced L3/4 protection
  • L7 application protection
  • Cost protection
  • Attack analytics and reporting
  • DDoS rapid response support

Protection Types

  • Volumetric Attacks: High-volume traffic floods
  • Protocol Attacks: Exploit protocol weaknesses
  • Resource Layer Attacks: Target application resources
  • Web Application Attacks: HTTP/HTTPS flood attacks

DDoS Protection Configuration

# Create DDoS Protection Plan az network ddos-protection create \ --name MyDdosProtectionPlan \ --resource-group MyResourceGroup \ --location eastus # Enable DDoS Protection on VNet az network vnet update \ --name MyVNet \ --resource-group MyResourceGroup \ --ddos-protection-plan /subscriptions/.../MyDdosProtectionPlan \ --ddos-protection true

Monitoring & Alerting

  • Attack metrics and analytics
  • Real-time attack notifications
  • Post-attack reports
  • Integration with Azure Monitor
  • Custom alerting rules

Private Access to Azure Services

Private Link

Azure Private Link enables you to access Azure PaaS services (like Azure Storage, SQL Database) and Azure-hosted customer/partner services over a private endpoint in your virtual network.

Private Link Components

  • Private Endpoint: Network interface for accessing services privately
  • Private Link Service: Your service behind Private Link
  • Private DNS Zone: DNS resolution for private endpoints
# Create private endpoint az network private-endpoint create \ --name MyPrivateEndpoint \ --resource-group MyResourceGroup \ --vnet-name MyVNet \ --subnet MySubnet \ --private-connection-resource-id /subscriptions/.../storageAccount \ --group-id blob # Create private DNS zone az network private-dns zone create \ --name privatelink.blob.core.windows.net \ --resource-group MyResourceGroup # Link DNS zone to VNet az network private-dns link vnet create \ --name MyDNSLink \ --resource-group MyResourceGroup \ --zone-name privatelink.blob.core.windows.net \ --virtual-network MyVNet \ --registration-enabled false

Service Endpoints

Service endpoints provide secure and direct connectivity to Azure services over the Azure backbone network.

Service Endpoint Benefits

  • Traffic stays on Azure backbone network
  • Optimal routing to Azure services
  • Simplified network configuration
  • No public IP required for services
  • Fine-grained access control

Supported Services

• Azure Storage
• Azure SQL Database
• Azure Cosmos DB
• Azure Key Vault
• Azure Container Registry
• Azure Event Hub
• Azure Service Bus
• Azure App Configuration
# Enable service endpoint on subnet az network vnet subnet update \ --name MySubnet \ --vnet-name MyVNet \ --resource-group MyResourceGroup \ --service-endpoints Microsoft.Storage Microsoft.Sql # Configure service endpoint policy az network service-endpoint policy create \ --name MySEPolicy \ --resource-group MyResourceGroup \ --location eastus # Add policy definition az network service-endpoint policy definition create \ --name AllowStorage \ --policy-name MySEPolicy \ --resource-group MyResourceGroup \ --service Microsoft.Storage \ --service-resources /subscriptions/.../storageAccounts/myaccount

Practice Questions

Azure Firewall

You need to inspect HTTPS traffic for threats and filter based on URLs. Which Azure Firewall SKU should you use?

A) Basic
B) Standard
C) Premium
D) Free
Correct! Azure Firewall Premium includes TLS inspection, IDPS signatures, and URL filtering capabilities required for HTTPS inspection.
Private Link

You want to access Azure Storage from your VNet without going over the public internet. What should you implement?

A) Private Endpoint
B) Service Endpoint
C) VPN Gateway
D) ExpressRoute
Correct! Private Endpoint provides a private IP address for accessing Azure Storage from your VNet over the Azure backbone network.
DDoS Protection

Your application is experiencing a volumetric DDoS attack. Which feature provides automatic mitigation?

A) DDoS Protection Basic
B) NSG Rules
C) Azure Firewall
D) Application Gateway WAF
Correct! DDoS Protection Basic provides always-on monitoring and automatic attack mitigation for volumetric attacks at no additional cost.
