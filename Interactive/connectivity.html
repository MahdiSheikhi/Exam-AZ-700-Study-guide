VPN Gateway

VPN Gateway Overview

Azure VPN Gateway enables you to create secure cross-premises connectivity between your Azure VNet and on-premises locations. It supports both site-to-site and point-to-site connections.

Key Features

High Availability

Active-Active and Zone Redundant options

Multiple VPN Types

Policy-based and Route-based VPNs

Multiple Protocols

IPsec, IKEv2, SSTP, OpenVPN

Scalability

Multiple SKUs for different performance needs

VPN Gateway topology showing site-to-site and point-to-site connections

Site-to-Site VPN

Site-to-Site VPN creates an encrypted connection between your on-premises network and Azure VNet over the internet.

Configuration Steps

  1. Create Virtual Network and Gateway Subnet
  2. Create Local Network Gateway
  3. Create VPN Gateway
  4. Create VPN Connection
  5. Configure On-premises VPN Device
# Create Local Network Gateway az network local-gateway create \ --name MyLocalGateway \ --resource-group MyResourceGroup \ --gateway-ip-address 203.0.113.1 \ --local-address-prefixes 192.168.0.0/24 \ --location eastus # Create VPN Connection az network vpn-connection create \ --name MyVPNConnection \ --resource-group MyResourceGroup \ --vnet-gateway1 MyVPNGateway \ --local-gateway2 MyLocalGateway \ --shared-key "YourSharedKey123!"

VPN Device Requirements

Requirement Specification
Public IP Static, Public IPv4
IKE Version IKEv1 or IKEv2
Encryption AES-256, SHA-256
Authentication Pre-shared Key

Point-to-Site VPN

Point-to-Site VPN allows individual clients to connect securely to your Azure VNet from anywhere.

Authentication Methods

RADIUS Authentication

Integrate with existing RADIUS servers

Certificate Authentication

Client certificates for authentication

Azure AD Authentication

Use Azure Active Directory credentials

# Configure Point-to-Site VPN az network vnet-gateway update \ --name MyVPNGateway \ --resource-group MyResourceGroup \ --address-prefixes 172.16.0.0/24 \ --client-protocol IkeV2 \ --aad-tenant https://login.microsoftonline.com/yourtenant \ --aad-audience 41b23e61-6c1e-4545-b367-cd054e0ed4b4 \ --aad-issuer https://sts.windows.net/yourtenant/

ExpressRoute

ExpressRoute provides private, dedicated network connectivity from your on-premises infrastructure to Azure. It offers higher reliability, faster speeds, and lower latencies than typical internet connections.

ExpressRoute Benefits

  • Private connectivity (not over internet)
  • High throughput and low latency
  • Layer 3 connectivity
  • 99.95% availability SLA
  • Global reach capabilities
  • Better security and compliance

ExpressRoute Models

  • ExpressRoute Direct: 10 Gbps or 100 Gbps direct connectivity
  • ExpressRoute Partner: Through connectivity provider
  • ExpressRoute Local: Cost-effective for data center proximity

ExpressRoute Peering Types

Private Peering

Access to Azure services in your VNet (IaaS, PaaS)

Microsoft Peering

Access to Microsoft services (Office 365, Dynamics 365)

Public Peering (Deprecated)

Replaced by Microsoft peering

# Create ExpressRoute circuit az network express-route create \ --name MyExpressRoute \ --resource-group MyResourceGroup \ --location eastus \ --bandwidth 100 \ --peering-location "Washington DC" \ --provider "Equinix" \ --sku-tier Standard # Configure private peering az network express-route peering create \ --name AzurePrivatePeering \ --circuit-name MyExpressRoute \ --resource-group MyResourceGroup \ --peering-type AzurePrivatePeering \ --primary-peer-subnet 192.168.1.0/30 \ --secondary-peer-subnet 192.168.2.0/30 \ --vlan-id 100 \ --peer-asn 65001

Azure Virtual WAN

Azure Virtual WAN is a networking service that brings together many networking, security, and routing functionalities into a single operational interface.

Virtual WAN Components

  • Virtual WAN Hub: Central connectivity point
  • Hub VNet Connection: Connect VNets to hub
  • Site-to-site VPN: Branch office connectivity
  • ExpressRoute: Dedicated connectivity
  • Point-to-site VPN: Remote user access
  • Secured Virtual Hub: Built-in security

Virtual WAN Benefits

  • Simplified network architecture
  • Automated hub-to-hub connectivity
  • Integrated security services
  • Global network optimization
  • Centralized management

Virtual WAN Architecture

Virtual WAN hub-and-spoke architecture with multiple connectivity options

# Create Virtual WAN az network vwan create \ --name MyVirtualWAN \ --resource-group MyResourceGroup \ --location eastus \ --type Standard # Create Virtual Hub az network vhub create \ --name MyVirtualHub \ --resource-group MyResourceGroup \ --location eastus \ --address-prefix 10.0.0.0/16 \ --vwan MyVirtualWAN

Practice Questions

VPN Gateway

You need to support 2 Gbps of VPN traffic with active-active mode. Which VPN Gateway SKU should you choose?

A) VpnGw2
B) VpnGw4
C) VpnGw3
D) VpnGw1
Correct! VpnGw4 supports up to 2.5 Gbps bandwidth and includes active-active mode capability.
ExpressRoute

You need to access Office 365 services through ExpressRoute. Which peering type should you configure?

A) Private Peering
B) Microsoft Peering
C) Public Peering
D) Global Peering
Correct! Microsoft peering provides access to Microsoft services including Office 365, Dynamics 365, and other SaaS services.
Virtual WAN

Your organization has multiple branch offices and wants simplified network management with built-in security. What should you implement?

A) Multiple VPN Gateways
B) ExpressRoute Circuits
C) Azure Virtual WAN
D) VNet Peering
Correct! Azure Virtual WAN provides a centralized hub for branch connectivity with integrated security and simplified management.
