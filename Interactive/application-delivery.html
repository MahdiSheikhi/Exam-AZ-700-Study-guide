Load Balancer Configuration

# Create public load balancer az network lb create \ --name MyLoadBalancer \ --resource-group MyResourceGroup \ --location eastus \ --sku Standard \ --public-ip-address MyPublicIP # Create backend pool az network lb address-pool create \ --name MyBackendPool \ --lb-name MyLoadBalancer \ --resource-group MyResourceGroup # Create health probe az network lb probe create \ --name MyHealthProbe \ --lb-name MyLoadBalancer \ --resource-group MyResourceGroup \ --protocol Http \ --port 80 \ --path /health # Create load balancing rule az network lb rule create \ --name MyLBRule \ --lb-name MyLoadBalancer \ --resource-group MyResourceGroup \ --protocol Tcp \ --frontend-port 80 \ --backend-port 80 \ --frontend-ip-name LoadBalancerFrontEnd \ --backend-pool-name MyBackendPool \ --probe-name MyHealthProbe

Distribution Algorithms

Hash-based Distribution

Uses 5-tuple hash (source IP, source port, destination IP, destination port, protocol)

Source IP Affinity

Routes traffic from same client IP to same backend instance

Application Gateway

Application Gateway is a Layer 7 load balancer that provides application delivery controller (ADC) functionality as a managed service. It offers various Layer-7 load balancing, routing, TLS offloading, and web application firewall functionalities.

Application Gateway Components

  • Frontend IP: Public or private IP for client connections
  • Listeners: Check for incoming requests on specific ports
  • Rules: Route requests to appropriate backend pools
  • Backend Pools: Target VMs or instances
  • HTTP Settings: Configure protocol, port, and cookie affinity
  • Health Probes: Monitor backend instance health

Key Features

  • SSL/TLS termination and end-to-end encryption
  • URL-based and path-based routing
  • Session affinity (sticky sessions)
  • Web Application Firewall (WAF)
  • Connection draining
  • Custom error pages
  • Request and response rewriting

Application Gateway SKUs

SKU Features Use Case
Standard Basic Layer-7 features General web applications
WAF Standard + WAF Applications requiring security
Standard_v2 Enhanced performance High-performance scenarios
WAF_v2 v2 + WAF High-performance + security
# Create Application Gateway az network application-gateway create \ --name MyAppGateway \ --resource-group MyResourceGroup \ --location eastus \ --capacity 2 \ --sku Standard_v2 \ --http-settings-protocol Http \ --frontend-port 80 \ --routing-rule-type Basic \ --servers 10.0.1.4 10.0.1.5

Azure Front Door

Azure Front Door is an application delivery network that provides global load balancing and site acceleration service for web applications. It offers Layer-7 capabilities for your application such as SSL offload, path-based routing, fast failover, and caching.

Front Door Key Features

  • Global HTTP/HTTPS load balancing
  • SSL offloading and end-to-end SSL
  • Application layer security with WAF
  • URL-based routing and redirection
  • Session affinity
  • Custom domains and certificate management
  • Health monitoring and automatic failover
  • Content caching for performance

Front Door vs Application Gateway

  • Scope: Front Door is global, App Gateway is regional
  • Routing: Front Door uses anycast, App Gateway uses regional load balancing
  • Performance: Front Door has edge presence, App Gateway is data center focused
  • Use Case: Front Door for global apps, App Gateway for regional apps

Front Door Architecture

Front Door global routing architecture with edge presence

# Create Front Door az network front-door create \ --name MyFrontDoor \ --resource-group MyResourceGroup \ --backend-address webapp1.azurewebsites.net \ --backend-address webapp2.azurewebsites.net # Add routing rule az network front-door routing-rule create \ --name MyRoutingRule \ --front-door-name MyFrontDoor \ --resource-group MyResourceGroup \ --frontend-endpoints DefaultFrontendEndpoint \ --route-type Forward \ --backend-pool DefaultBackendPool

Traffic Manager

Traffic Manager is a DNS-based traffic load balancer that enables you to distribute traffic optimally to services across global Azure regions, while providing high availability and responsiveness.

Traffic Manager Benefits

  • DNS-based global traffic distribution
  • Multiple traffic routing methods
  • Endpoint health monitoring
  • Automatic failover capabilities
  • Performance optimization
  • Geographic distribution control

Routing Methods

Performance

Route to endpoint with lowest latency

Priority

Primary/backup failover model

Weighted

Distribute traffic based on weights

Geographic

Route based on user location

Subnet

Route based on IP address range

Traffic Manager Configuration

# Create Traffic Manager profile az network traffic-manager profile create \ --name MyTrafficManagerProfile \ --resource-group MyResourceGroup \ --routing-method Performance \ --unique-dns-name myapp-dns # Add endpoints az network traffic-manager endpoint create \ --name WebApp1 \ --profile-name MyTrafficManagerProfile \ --resource-group MyResourceGroup \ --type azureEndpoints \ --target-resource-id /subscriptions/.../webapp1 \ --endpoint-status Enabled az network traffic-manager endpoint create \ --name WebApp2 \ --profile-name MyTrafficManagerProfile \ --resource-group MyResourceGroup \ --type azureEndpoints \ --target-resource-id /subscriptions/.../webapp2 \ --endpoint-status Enabled

Practice Questions

Load Balancer

You need to distribute UDP traffic across multiple VMs with the lowest possible latency. Which service should you use?

A) Azure Load Balancer
B) Application Gateway
C) Front Door
D) Traffic Manager
Correct! Azure Load Balancer operates at Layer 4 and supports UDP traffic with ultra-low latency, making it ideal for this scenario.
Application Gateway

You need to route web traffic to different backend pools based on URL paths (/api, /images, /). Which feature should you implement?

A) Load Balancer rules
B) Path-based routing
C) Traffic Manager profiles
D) Front Door rules
Correct! Application Gateway's path-based routing allows you to route requests to different backend pools based on URL paths.
Global Load Balancing

You have web applications deployed in multiple Azure regions and want to route users to the closest region for optimal performance. Which service should you use?

A) Application Gateway
B) Traffic Manager (Priority)
C) Load Balancer
D) Traffic Manager (Performance)
Correct! Traffic Manager with Performance routing method automatically routes users to the endpoint with the lowest latency.
